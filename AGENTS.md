# Repository Agent Instructions

This repository contains public Codex skills and synthetic examples for thermal-machinery and gas-turbine modeling workflows.

## Working Rules

- Do not add private project files, local paths, personal identifiers, unpublished reports, or proprietary data.
- Keep examples synthetic, reduced, or explicitly public.
- Do not add Simulink models, MATLAB data files, office documents, PDFs, archives, or images unless a maintainer explicitly approves them for public release.
- Use `scripts/validate_release.py` before claiming the repository is release-ready.
- Treat release checks as repository packaging checks, not engineering validation.

## Skill Editing

- Keep `SKILL.md` concise.
- Put detailed procedures in `references/`.
- Prefer reusable workflow rules over project-specific details.
- Keep units, assumptions, validation gates, and source provenance explicit.
- Avoid claiming a model is validated unless the evidence type is stated.
