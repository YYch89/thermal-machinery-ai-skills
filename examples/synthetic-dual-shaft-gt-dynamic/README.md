# Dual-shaft free-power-turbine gas turbine model

This package contains a staged MATLAB/Simulink public example for a
generic 16 MW islanded DC microgrid gas turbine generator set.

This is a synthetic example prepared for demonstrating the
`gas-turbine-ai-modeling` workflow. It uses only synthetic public data and
does not commit generated Simulink binaries. The Simulink models are
generated by scripts rather than committed as `.slx` files.

Model status: exploratory/reduced.

Implemented stages:

1. MATLAB design-point data source.
2. Map audit and wrapper specification.
3. Rated steady map-wrapper closure.
4. Dynamic initial-condition registry and plant residual check.
5. Exploratory Simulink closed-loop model.
6. Native-block component Simulink model.
7. Power-turbine speed control check.

All dynamic initial values and map scaling constants should trace back to
the MATLAB design-point data source or to a recorded trim result.

## Current rated point

- Application: generic islanded DC microgrid generation.
- Architecture: dual-shaft free power turbine.
- Rated DC power: 16 MW.
- Power turbine speed target: 3000 rpm.
- Gas-generator speed reference: 12000 rpm.
- Combustor outlet design temperature: 1440 K.
- Compressor pressure ratio assumption: 14.
- Fuel: generic liquid fuel, exploratory LHV 42.7 MJ/kg.
- Design air flow: 50.930914 kg/s.
- Design fuel flow: 1.251298 kg/s.

## Key files

- `src/build_gt_model_data.m`: single source for requirements, assumptions,
  design point, map scaling, dynamics, limits, and initial values.
- `src/audit_gt_maps.m`: read-only synthetic compressor-map and turbine-map
  audit.
- `src/evaluate_gt_steady_rated.m`: rated steady map-wrapper closure.
- `src/evaluate_gt_dynamics.m`: reduced dynamic plant state equations.
- `src/evaluate_gt_components.m`: componentized MATLAB reference for
  numeric equivalence checks.
- `docs/component_contracts.md`: component interfaces, equations, states,
  and residual checks.
- `scripts/build_gt_simulink_model.m`: reproducibly builds
  `simulink/GT_DualShaft_Dynamic_Exploratory.slx`.
- `scripts/run_simulink_closed_loop.m`: fixed-step closed-loop validation.
- `scripts/build_gt_component_native_model.m`: reproducibly builds
  `simulink/GT_DualShaft_Component_Native.slx` from native Simulink blocks.
- `scripts/run_component_native_closed_loop.m`: fixed-step validation for
  the native-block component model.

## Smoke Checks

Run from MATLAB or the package root:

```matlab
runtests('tests')
run('scripts/build_gt_simulink_model.m')
run('scripts/run_simulink_closed_loop.m')
run('scripts/build_gt_component_native_model.m')
run('scripts/run_component_native_closed_loop.m')
```

The current Simulink smoke-check case is a rated-load rejection from
16.0 MW to 15.2 MW DC at 2 s. Both Simulink models use fixed-step `ode4`
with 0.01 s step size.

Expected checks:

- Design-point pressure, temperature, fuel-air-ratio, and shaft-power
  consistency checks pass.
- Rated steady map-wrapper closure reproduces the design air flow.
- Dynamic initial residuals are near zero at the design point.
- Closed-loop load rejection keeps the compressor map valid and returns
  power-turbine speed close to the reference.

The native-block compressor map uses a dense 2-D Lookup Table generated
from the synthetic public pressure-ratio/relative-speed formula, plus a
separate 2-D validity table. This keeps the example self-contained while
still exercising the map-scaling and map-validity workflow.

## Important next modeling decision

The 16 MW design point uses a combustor outlet design temperature of
1440 K. This is not an absolute temperature boundary. Continuous and
transient temperature limits still need to be provided or assumed before
positive load pickup, temperature limiting, or overload response can be
checked. Choose one of these before tuning further:

1. Derate nominal operation below 1440 K and reserve temperature margin.
2. Keep 16 MW as the design point and provide continuous/transient T3
   limits for positive load pickup.
3. Provide a higher transient temperature allowance or a detailed
   temperature limiter schedule.
